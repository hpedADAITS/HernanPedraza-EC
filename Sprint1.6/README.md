# Technical Specification Document: Automatic Documentation Generator for Java Projects

The goal of this project is to develop a fully modular, self-contained application that automatically analyzes Java source code, generates comprehensive technical documentation, and delivers it in structured formats (Markdown and PDF), while producing UML diagrams using PlantUML.

All components are containerized using Docker. The application runs with a Node.js backend (REST API), React frontend (with TailwindCSS UI), and leverages locally deployed AI models (via LMStudio) to enhance documentation quality in an offline, privacy-preserving manner.

---

## 7. Key Enhancements Implemented (Beyond Minimum Requirements)

| Enhancement | Description |
|----------|-------------|
| **Project Analysis Statistics** | Displays key metrics: total classes, methods, packages, file count, average class size. |
| **Filtering by Package or Class** | Users can filter results to view only specific packages (`com.example.user`) or classes (e.g., `User`, `Order`). |
| **Version Comparison Support (Partial)** | Compares two project versions side-by-side using diff logic on file structure and method changes. Only supports Java files with identical paths; no semantic diffing yet. |
| **AI Recommendations** | The EXPANDER model generates design suggestions, e.g., “Consider adding error handling in `login()`” or “This class may benefit from refactoring into a service layer.” |
| **Improved Frontend Accessibility & UX** | Full ARIA support, keyboard navigation, responsive layout with Tailwind, clear loading/error states. |
| **Structured AI Tool Call Validation** | All AI-generated responses are validated using JSON schema to ensure tool calls were executed and outputs are structured properly. |

## 10. Notes on Assumptions & Limitations

- **Java Parser Accuracy**: Basic syntax parsing is sufficient for class/method detection; complex generics, annotations, or lambdas may require more advanced AST tools (e.g., Eclipse JDT).
- **Tool Call Reliability**: `grep` patterns are simple and may miss edge cases. Future work: use AST-based Java parser.
- **Performance**: Large projects (>100 files) may take several minutes to process due to AI model inference time.
- **AI Output Consistency**: Descriptions generated by EXPANDER depend on prompt quality; no guarantee of correctness or completeness.

> All core functionality must be implemented and tested. The system must be production-ready for small-to-medium Java projects (10–50 files). Larger projects require optimization in parsing speed and memory usage.
